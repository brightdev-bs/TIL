## 객체지향 프로그래밍  ##
객체지향이론의 기본 개념은 '실제 세계는 사물(객체)로 이루어져 있고, 발생하는 모든 사건들은 사물간의 상호작용에 의해 발생한다는 점에 기반한다.<br><br>
* **유지보수**, **중복된 코드의 제거** 그리고 **재사용성**의 관점에서 객체지향을 이해할 필요가 있다.*

--------
#### 클래스와 객체  ####
클래스란?<br>
 -> 객체를 정의해놓은 것.<br>
 -> 객체의 설계도 또는 틀. <br><br>

객체란? <br>
 -> 실제로 존재하는 것. <br>
 -> 클래스에 정의된 내용대로 메모리에 생성된 것. <br><br>

객체 배열<br>
 -> 기본형 배열과 달리 객체 배열은 각 배열에 객체의 주소를 저장한다. <br>

객체 배열도 다른 모든 배열과 마찬가지로 같은 타입의 객체만 저장이 가능하다. <br>
여러 종류의 객체를 하나의 배열에 저장하기 위해선 '다형성'을 이용할 수 있다.

--------
#### 변수와 메서드  ####
변수는 클래스변수, 인스턴스변수, 지역변수 총 3가지가 있다. **변수의 선언된 위치**는 변수의 종류를 파악하는 중요한 요소이다.

1 ) 인스턴스 변수<br>
 -> 클래스 영역에 선언되며, 클래스의 인스턴스를 생성할 때 만들어진다.<br>
 -> 인스턴스마다 고유한 상태를 유지해야하는 속성의 경우, 인스턴스 변수로 선언한다.<br>

2 ) 클래스 변수 <br>
 -> 인스턴스 변수 앞에 static을 붙이면 클래스 변수가 된다.
 -> 모든 인스턴스가 공통된 저장공간(변수)을 공유하게 된다.

3 ) 지역변수
 -> 메서드 내에 선언되어 메소드 내에서만 사용가능하며, 메소드가 종료되면 소멸된다. 

--------
#### 메서드  ####
메소드란?<br>
 -> 특정 작업을 수행하는 일련의 문장들을 하나로 묶은 것. <br>
 -> 메서드에 넣을 값(입력)과 반환하는 결과(출력)만 알면 사용이 가능하다. <br>
 -> 메서드의 선언부는 후에 변경사항이 발생하지 않도록 신중히 작성해야 한다. (이름만으로도 메서드의 기능을 알 수 있도록 함축적이면서도 의미 있는 이름을 지어야 한다.) <br><br>

**메소드의 장점**<br>

1. 높은 재사용성 : 한 번 만들어 놓으면 몇 번이고 메서드를 호출하여 사용할 수 있다.
2. 중복된 코드의 제거 : 프로그램을 작성하다보면, 같은 내용의 문장들이 여러 곳에 반복해서 나타나는데, 이런 반복되는 문장들을 하나의 메서드로 작성해 놓으면 메소드 한 문장으로 대체할 수 있다.
3. **프로그램의 구조화** : 큰 규모에서는 문장들을 작업단위로 나눠서 여러 개의 메서드에 담아 프로그램의 구조를 단순화시키는 것이 필수적이다.<br>
  -> 예제 코드 <br>
    
		public static void main(String args[]) {
			int[] numArr = new int[10];
			initArr(numArr); // 1. 배열을 초기화한다.
			printArr(numArr); // 2. 배열을 출력한다.
			sortArr(numArr); // 3. 배열을 정렬한다.
			printArr(numArr); // 4. 배열을 출력한다.
		}
    	
 <br> 
> 처음에 프로그램을 설계할 때 내용이 없는 메서드를 작업단위로 만들어 놓고, 하나씩 완성해가는 것도 프로그램을 구조화하는 좋은 방법이다.
 <br><br>

**매개변수의 유효성 검사**<br>
&nbsp;&nbsp; 메서드의 구현부를 작성할 때, 제일 먼저 매개변수의 값이 적절한 것인지 확인해야 한다. 


**기본형 매개변수와 참조형 매개변수**<br>
&nbsp;&nbsp;
매개변수의 타입에 따라 주소 전달과 값 복사가 결정된다. 타입이 기본형일 때는 기본형 값이 복사되지만, 참조형이면 인스턴스의 주소가 복사된다.

**클래스 메서드와 인스턴스 메서드의 사용**<br>
&nbsp;&nbsp; 인스턴스 메서드는 인스턴스 변수와 관련된 작업을 해서 수행을 위해 그 변수가 필요할 때 인스턴스 메서드를 사용한다. <br>
&nbsp;&nbsp; 클래스 메소드는 인스턴스와 관계없는 메서드를 클래스 메서드로 정의한다. (공통으로 사용되는 것)

#### 오버로딩 ####
오버로딩이란?
&nbsp;&nbsp; 한 클래스 내에 같은 이름의 메서드를 여러 개 정의하는 것.<br>
> 가변인자를 사용한 메서드는 오버로딩하지 않는 것이 좋다.
