# std::array와 std::vector

std::array는 고정된 배열을 사용할 때 유용하다. 그러나 몇 가지 단점이 있다.

1. 크기가 컴파일 시간에 결정되어야 한다.
2. 크기가 고정되어 있다.
3. 스택을 사용하기 때문에 메모리 할당을 할 수 없다.

---

std::array의 가장 큰 문제점이 고정된 크기 문제를 vector에서 해결할 수 있다.

```cpp
// 크기가 0인 벡터 선언
std::vector<int> v;

// 크기가 8인 벡터 선언
std::vector<int> vec(8);

// 지정한 값들로 이루어진 크기가 3인 벡터 선언
std::vector<int> vec = {1,2,3}

//크키가 10이고, 모든 원소가 3으로 초기화된 벡터 선언
std::vector<int> vec(10,3);
```

벡터의 선언 이후 새로운 원소를 추가하려면 2가지 함수를 이용한다.

1. push_back() : 벡터의 맨 마지막에 새로운 원소를 추가
2. insert() : 첫 번째 인자로 받은 값에 삽입

push_back()과 insert() 함수의 단점은 추가할 원소를 임시로 생성한 후에 벡터 버퍼 내부로 복사 또는 이동을 수행한다는 점이다. 

❗이러한 단점은 새로운 원소가 추가될 위치에서 해당 원소를 생성하는 방식으로 해결할 수 있다.

```cpp
emplace_back(); 
emplace()
```

---

원소 제거를 위해 2가지 함수를 이용할 수 있다.

1. pop_back() : 맨 마지막 원소를 제거, 벡터 크기 1 줄어듬
2. erase() : 인자를 하나 받아 해당 위치 원소 제거

                  : 시작 위치와 끝을 나타내는 인자를 받아 그 사이의 원소 제거

```cpp
vec.pop_back();
// {0,1,2,3,4}

vec.erase(vec.begin());
// {1,2,3,4}

vec.erase(vec.begin(), vec.begin() + 3)
// { 4 }
```

---

유용한 std::vector 멤버 함수

1. clear() : 모든 원소를 제거
2. reserve() : 벡터에서 사용할 용량을 지정
3. shrink_to_fit() : 여분의 메모리 공간 해제 / 벡터 용량 = 벡터 크기

                         ​                         :  벡터 크기가 더 이상 변경되지 않을 때 사용됨.
