# ICT Intership

해커랭크에서 12시간 동안 코딩 테스트를 봤다.

처음 보는 코딩테스트였고, 난이도가 어렵지 않을까 걱정을 많이 했는데 생각보다 쉬웠던 것 같다.

알고리즘 공부를 한 지 2주 정도 된 것 같은데 이번 시험을 통해 어떻게 공부해야 되는지 어느 정도 감이 온 것 같다.

오늘 시험에서 1번 2번 문제는 오름차순으로 정렬하면 쉽게 풀 수 있는 문제였다.

3번 문제는 운영체제의 cpu 스케줄링 알고리즘이 떠오른 문제였다. 대기 시간을 최대로 줄이기 위해서 배열을 오름차순으로 정렬한 후, 최소가 되는 방식으로 풀이 했다.

 → 4개 정도 테스트 케이스를 만족시키기 못하였다.

4번 문제는 재귀 문제였는데 두 가지 수가 주어지고 n이라는 수를 m이하의 숫자들로 표현하면 몇 개를 표현할 수 있는지에 대한 문제였다. 재귀로 코딩했고, 시간 최적화하라는 안내가 떠서 메모이제이션을 이용해 풀이했다.

 → 메모이제이션을 했음에도 2개는 시간 초과, 2개는 난수가 나왔다.

5번 문제는 트리 구조를 표현하는 문제였는데 화살표가 아닌 '('를 이용하여 왼쪽 자식 오른쪽 자식을 표현해야 했다. 생각보다 이 작업이 오래걸렸고, string으로 문제가 나와 더 어려웠다. 나머지는 순환이 생기는지, 루트노드가 여러개 있는지, 자식 노드가 2개 초과했는지 등을 확인하는 문제였고, 순환과 루트노드, 자식 노드 초과 등은 코딩했으나, 틀린 것 같다..

이번 시험을 통해 코딩 테스트를 어떻게 준비해야 하는지 감이 조금 온 것 같다.

재귀, 동적 다이나믹 프로그래밍, string, 분할 정복 파트가 나의 큰 약점인 것 같다.
