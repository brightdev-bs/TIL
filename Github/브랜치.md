# 브랜치

제품을 출시하고 나면, 고객사마다 추가로 요구하는 내용이 다르다.

고객들마다 요구하는 내용을 반영하다 보면 프로그램도 조금씩 달라질 것이다.

이에 맞춰 제품 설명서도 달라져야 하는데 이 때 사용하는 것이 브랜치이다.

---

git init을 하면 기본적으로 master라는 브랜치가 만들어진다.

사용자가 commit을 할 때마다 master 브랜치는 최신 커밋을 가리킨다.

 → 브랜치는 커밋을 가리키는 포인터

여기서 새 브랜치를 만들면 기존에 저장한 파일을 master브랜치에 그대로 유지하면서 파일을 수정하거나 새로운 기능을 추가할 수 있다.

---

깃에서 브랜치를 만들거나 확인할 때는 다음의 명령어를 사용한다.

```c
git brach
git branch mango
```

다른 브랜치로 사용을 옮길려면 다음의 명령어를 사용한다.

```c
git checkout mango
```

브랜치 사이의 차이점을 알아보자.

마침표 왼쪽에 있는 브랜치를 기준으로 오른쪽 브랜치를 비교한다.

```c
git log master..mango
```

브랜치를 병합할 때는 다음의 명령어를 사용한다

 - master브랜치에서 mango브랜치를 병합하고 싶을 때는 master브랜치로 이동 후 다음의 명령어를 사용한다.

```c
git merge mango
```

사용하지 않는 브랜치를 삭제할 때는 다음의 명령어를 사용한다.

 -영구적인 삭제가 아니라, 같은 이름의 브랜치를 다시 만들면 예전 내용을 다시 볼 수 있다.

```c
git branch -d mango
```

깃에서는 줄 단위로 변경 여부를 확인한다.

따라서 같은 파일 이름을 가지고 같은 줄을 수정했을 때, 충돌이 발생한다.

이럴 때는 양쪽 브랜치의 내용을 참고하면서 직접 내용을 수정해야 한다.
